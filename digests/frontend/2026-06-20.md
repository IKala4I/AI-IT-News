## ⚛️ React Core

### React Compiler ported to Rust — all 1,725 tests passing
The big story of the week. Joe Savona's [PR #36173](https://github.com/react/react/pull/36173) merged June 9 and landed in active integration this week. The Rust compiler passes every existing fixture with snapshot parity to the TypeScript version.

**Why it matters:** 3× faster as a Babel plugin drop-in; ~10× faster for raw transformation logic. Linked directly into Turbopack it already cuts compile time by >40%. Meta is running it in production — Instagram and the Quest Store report 12% faster initial loads and 2.5× faster interactions. SWC and Bun are already pulling it via git dependency; crates.io publication is next.

**Breaking changes / migration:** None yet for app developers. The TypeScript compiler path stays in place while the Rust version stabilises. Framework authors (SWC/OXC integrations) should watch the `react_compiler_oxc` and `react_compiler_swc` crates.

- [PR #36173](https://github.com/react/react/pull/36173)
- [Hacker News discussion](https://news.ycombinator.com/item?id=48473662)
- [Joe Savona on X](https://x.com/en_JS/status/2031606726689173846)

## 📱 React Native & Expo

### React Native 0.86 — edge-to-edge & zero breaking changes
Released June 11 (blog post and community discussion active this week). Second consecutive release with no user-facing breaking changes.

**Key changes:**
- Full edge-to-edge support for Android 15+ (backward-compatible via new `EdgeToEdge` component)
- React Native DevTools: light/dark-mode emulation, custom renderer operation tracking
- Android: TextInput extended autofill types, hardware play/pause events, LogBox dismiss via back button
- iOS: non-blocking packager validation, privacy manifest declarations, multipart request interceptor API
- Metro bumped to `^0.84.2`; Hermes compiler deps updated

**Why it matters:** The no-breaking-changes streak is intentional — the team is signalling that upgrade friction is a first-class concern. Expo SDK 56 is expected to adopt 0.86 shortly.

- [Release notes](https://github.com/facebook/react-native/releases/tag/v0.86.0)
- [Blog post](https://reactnative.dev/blog/2026/06/11/react-native-0.86)

### Worklets 0.10.0 — Bundle Mode goes stable (June 19)
The `worklets` package (spun out of react-native-reanimated) reached its first stable feature milestone.

**Key changes:**
- Bundle Mode is now **stable and recommended** (no longer experimental)
- New `importForwarding` API replaces `workletizableModules` config
- Native serialization support for `ArrayBufferView` objects — avoids serializing typed arrays through JS bridge
- Various bug fixes for Android builds and source maps

**Why it matters:** Bundle Mode dramatically reduces worklet overhead in large apps. If you used `workletizableModules`, migrate to `importForwarding` before the old API is removed.

**Breaking change:** `workletizableModules` is deprecated; `importForwarding` is the new API.

- [Worklets 0.10.0 release](https://github.com/software-mansion/react-native-reanimated/releases/tag/worklets-0.10.0)

## ▲ Next.js / Vercel

### Vercel Ship 2026 — agentic infrastructure push (June 17, London)
~2,500 attendees; Guillermo Rauch framed Vercel's new direction as "where coding agents deploy software." Next.js itself had no stable release at the event — the focus was entirely on the agent layer above it.

**Announcements:**

| Product | Status | What it is |
|---|---|---|
| **eve** | Open source | TypeScript agent framework — "Next.js for agents." Durable execution, sandboxed compute, built-in observability, human-in-the-loop approvals. Used internally for v0. |
| **Vercel Connect** | Beta | Scoped, short-lived credentials for agents to call Slack, GitHub, etc. |
| **Vercel Services** | GA July 1 | Microservices as first-class citizens; internal service-to-service calls without public exposure. |
| **Vercel Agent** | Private beta | Autonomous production monitoring — detects anomalies and opens PRs. |
| **Enterprise Managed Users** | Private beta | Identity & access management for enterprise Vercel orgs. |

**Why it matters for React/Next.js devs:** The Next.js 16.2 AI features (browser log forwarding, agent-ready `create-next-app`, experimental Agent DevTools) that shipped in March now have a full deployment target. The canary track this week added `useExperimentalReact` config (canary.53) which allows opt-in React feature flags per-app.

### Zod compiler tooling gaining traction
Two community tools are pulling attention as build-time schema compilation alternatives to Zod's runtime JIT:

- **zod-aot**: Compiles Zod schemas to plain JS validation functions at build time — claimed 2–64× faster than runtime validation.
- **zod-compiler** (by @gajus): Scans imports from `"zod"`, detects exported schemas, rewrites calls — transparent to tRPC, Hono, React Hook Form consumers.

**Why it matters:** If you're validating at the edge or in hot paths, moving schema compilation to build time removes a meaningful runtime cost with zero API changes.

- [zod-compiler GitHub](https://github.com/gajus/zod-compiler)
- [zod-aot dev.to post](https://dev.to/wakita181009/i-made-zod-64x-faster-by-compiling-schemas-at-build-time-3e1a)
