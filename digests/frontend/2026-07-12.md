## ⚛️ React Core

### React Compiler: two more real-world memoization gotchas surface

Following last week's bundle-size/correctness debate, This Week In React #289 (July 8) flags two concrete breakage patterns people are hitting in production: the Compiler's auto-memoization conflicting with React Hook Form's `watch()`, and Chart.js event handlers going stale because the Compiler doesn't refresh their closures the way manual `useCallback` deps would.

**Why it matters:** two specific, testable cases to add to your Compiler-adoption checklist if you're running `react-hook-form` or `chart.js` alongside the Compiler.

- [This Week In React #289](https://thisweekinreact.com/newsletter/289)

### react.dev: Suspense boundary docs clarified

React Status #482 (July 10) flags a documentation update clarifying exactly what triggers a Suspense boundary to show its fallback, with new live/interactive examples. No code or behavior changes — docs-only.

## 📱 React Native & Expo

### React Native v0.87.0-rc.0 — the "Marie Kondo" release ⭐⭐

First release candidate of the 0.87 line (July 8), a deliberate legacy-cleanup release rather than a features release:

- **Removes deprecated legacy-architecture APIs**: old `StatusBar` `backgroundColor`/`translucent` props, boolean `keyboardShouldPersistTaps`, `NativeMethods`/`NativeMethodsMixin` TS types, and assorted legacy headers (e.g. `React/RCTTextView.h`)
- **Raises the minimum Node.js version to ≥ 22.13.0**
- **Android**: moves to AGP v9, `compileSdk` 37, requires Kotlin 2.0+
- Adds `ACCESS_LOCAL_NETWORK` permission support, a new `react-native/asset-utils` package, expanded `MapBuffer` entry types, `textAlign: 'start'/'end'`, ArrayBuffer support in C++/Obj-C TurboModules, CSS Flexbox §4.5 automatic minimum sizing
- DevTools gains light/dark mode emulation and new custom performance-trace tracks

**Breaking changes (explicit):** legacy-architecture API removals, Node ≥22.13.0 requirement, and the Android AGP v9/Kotlin 2.0+ bump are the headline migration items — verify exact wording against the release notes before upgrading a large app.

**Why it matters:** this is a pre-release (RC, not for production), but it signals what 0.87 stable will require — start checking your Node/Kotlin/AGP versions now if you plan to track RN closely.

- [React Native v0.87.0-rc.0 release notes](https://github.com/facebook/react-native/releases/tag/v0.87.0-rc.0)

### React Navigation 8.0 — July progress report: Suspense-enabled navigation

Published July 8. Headline change: React Navigation is now **Suspense-enabled** — navigation state updates are compatible with concurrent rendering, and navigation actions wrap in transitions where possible so the current screen stays visible instead of falling back to a loading state mid-navigation. Marked **experimental**, and currently **static-config only**.

**Why it matters:** if you're on React Navigation and adopting Suspense-for-data-fetching patterns, this closes a long-standing gap where navigating away could interrupt in-flight Suspense boundaries.

- [React Navigation 8.0 — July Progress Report](https://reactnavigation.org/blog/2026/07/08/react-navigation-8.0-july-progress/)

### Reanimated: Worklets 0.10.2 (July 6)

Small patch: fixes script-buffer reading for paths starting with `/`, and removes error handling that incorrectly fired when serialization creation failed. No breaking changes.

- [Reanimated releases](https://github.com/software-mansion/react-native-reanimated/releases)

### Software Mansion: Kotlin Multiplatform vs. React Native benchmarks

Via React Status #482 — a benchmark comparison from the Reanimated team's own org: KMP wins on Android, RN stays competitive on iOS. Useful data point if your team is evaluating KMP as an RN alternative for a new cross-platform project.

- [React Status #482](https://react.statuscode.com/issues/482)

### Expo: sign in with passkeys (July 6)

Expo account holders (expo.dev dashboard/CLI login, not app-level auth) can now register a passkey as a login method via expo.dev/settings#sign-in-methods, replacing password+OTP two-factor with a single stronger factor. Not available for SSO-managed accounts. This is about securing your Expo account, not an auth feature for apps you build with Expo.

- [expo.dev/changelog/sign-in-with-passkeys](https://expo.dev/changelog/sign-in-with-passkeys)

## ▲ Next.js / Vercel

### Vercel acquires Better Auth ⭐⭐

July 7 — Vercel has acquired **Better Auth**, the open-source TypeScript auth library (4.7M+ weekly downloads, 850+ contributors); its founder and core team join Vercel. The library stays free, MIT-licensed, framework-agnostic, and under its existing name/governance — no forced migration.

**Why it matters:** Better Auth is widely used in React/Next.js apps; Vercel is explicit that it remains portable (not locked to Vercel hosting). Longer-term, it's positioned as the foundation for "Agent Auth" — scoped, revocable identities for AI agents acting on a developer's behalf.

**Breaking changes:** None.

- [vercel.com/blog/vercel-acquires-better-auth](https://vercel.com/blog/vercel-acquires-better-auth)

### Vercel Agent: an agent you can let near production ⭐⭐

July 8 — Vercel Agent now lives in the Dashboard (plus GitHub/CLI integration) and can autonomously investigate production incidents, answer project questions, and propose fixes pending human approval. Demoed going from alert to rollback in under 3 minutes.

**Security model:** runs under a distinct `vercel-agent` identity for auditability; read-only by default, gains scoped permissions only after explicit plan approval (revoked immediately after); generated code executes in isolated Vercel Sandbox (Firecracker microVMs), never directly against production.

**Why it matters:** automates first-pass triage (logs, metrics, PR performance review, build debugging, cost analysis) for teams running Next.js on Vercel — reduces on-call load, though fixes still require human sign-off.

**Breaking changes:** None — new opt-in feature rolling out to Pro/Enterprise teams.

- [vercel.com/blog/vercel-agent](https://vercel.com/blog/vercel-agent)

### Next.js WebSocket RFC

Proposes `NextResponse.upgrade()` for first-class WebSocket support directly in Route Handlers, removing the need for a separate WS server alongside a Next.js app. Still an RFC, not shipped.

- [This Week In React #289](https://thisweekinreact.com/newsletter/289)

### Vinext 1.0 beta — Cloudflare's Next.js-on-Vite fork

Cloudflare's fork that runs Next.js on top of Vite instead of Webpack/Turbopack is nearing production readiness, with faster build times cited as the main draw. Worth watching if your Next.js app targets Cloudflare's platform.

- [This Week In React #289](https://thisweekinreact.com/newsletter/289)

Routine, low-signal canary progress — no distinct feature announcements in this batch, just continued incremental work toward 16.3 stable (building on the Instant Navigation hardening covered last week).

## 🧰 Libraries & Tooling

### Reverse-engineering ChatGPT's web app stack

A widely-shared technical teardown of OpenAI's ChatGPT web client, cross-linked by both newsletters this week: React Router v7 (migrated off Next.js), TanStack Query, Radix UI primitives, and ProseMirror for the rich-text editing surface.

**Why it matters:** a rare look at framework/library choices at extreme scale from a team with no vendor obligation to any one stack — useful signal for teams debating Next.js vs. React Router for a large, latency-sensitive app.

- [This Week In React #289](https://thisweekinreact.com/newsletter/289) · [React Status #482](https://react.statuscode.com/issues/482)

### TypeScript 7.0 released — ~10x faster Go-ported compiler

The full rewrite of the TypeScript compiler and language service in Go (the "Corsa" project) has shipped. Headline claim is roughly a 10x speedup in type-checking and project-load times, benefiting `tsc`, editor tooling, and CI type-check steps alike.

**Why it matters:** faster feedback loop for any nontrivial React/RN codebase's type-checking. Both newsletters note that some frameworks/toolchains are still recommending TypeScript 6.x while they finish compatibility work — check your framework's guidance before forcing the upgrade project-wide.

- [This Week In React #289](https://thisweekinreact.com/newsletter/289) · [React Status #482](https://react.statuscode.com/issues/482)

### ES2026 officially approved by Ecma

Adds `Array.fromAsync()`, `Error.isError()`, and `Map.getOrInsert()` to the language, among other additions. Also flagged in the same roundup: the IETF's new HTTP `QUERY` method standard (a GET-with-a-body method for cacheable, complex read queries).

**Why it matters:** `Array.fromAsync()` and `Map.getOrInsert()` are small but genuinely useful additions for everyday async/data-fetching code once runtime support lands.

- [This Week In React #289](https://thisweekinreact.com/newsletter/289)
