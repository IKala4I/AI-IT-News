## 📱 React Native та Expo

**Expo [оголосили деприкейт](https://expo.dev/blog/moving-away-from-expo-vector-icons) `@expo/vector-icons`** — обгортка була потрібна, поки `react-native-vector-icons` не вмів нормально працювати в Expo Go й OTA-апдейтах. Тепер scoped-пакети `@react-native-vector-icons/*` самі інтегруються з `expo-font` і працюють в Expo Go, dev-білдах і проді без жодної обгортки.

Головна причина мігрувати вже зараз — бандл: у тестовому застосунку Expo сама лише зміна імпортів і залежностей дала **-4MB**, бо стара обгортка тягла всі шрифти іконок одразу, навіть якщо юзаєш один-два набори.

```bash
npx @react-native-vector-icons/codemod
npx expo doctor
```

⚠️ **Міграція:** `@expo/vector-icons` і non-scoped `react-native-vector-icons` йдуть під деприкейт — переходь на `@react-native-vector-icons/*` через кодмод. [Деталі](https://expo.dev/blog/moving-away-from-expo-vector-icons)

Якщо в проєкті купа іконок — це реальна економія бандла за 10 хвилин роботи кодмода. Дедлайну поки нема, але простіше зробити зараз, поки апка невелика, ніж мігрувати щось критичне пізніше.

#expo

**react-native-screens [зарелізили 4.26](https://github.com/software-mansion/react-native-screens/releases/tag/4.26.0) зі стабільним Tabs API** — з experimental статусу перейшли до продакшен-готового: переписали внутрішню cross-platform реалізацію з observer registry, додали `programmatic-native` варіант action origin і інтеграцію з `ScrollViewMarker` для скрол-ефектів на iOS та Android.

Також у релізі:
• experimental standalone FormSheet для обох платформ
• розширене Native Stack menu API — вкладені меню, іконки, групування пунктів на Android

⚠️ **Міграція:** версія вимагає React Native 0.84+, а в Stack v5 header items перейменували `label`/`key` на `title`/`id`. [Реліз](https://github.com/software-mansion/react-native-screens/releases/tag/4.26.0)

Якщо вже юзаєш experimental Tabs — оновлюйся, стабільний API означає менше сюрпризів при апдейтах. Якщо ще на RN нижче 0.84 — спочатку підтягни RN, а вже потім цю версію.

## 🧰 Бібліотеки й тулінг

**Margelo [переписали Rive React Native SDK](https://blog.margelo.com/rewriting-rive-react-native-with-nitro-modules) на Nitro Modules замість старого bridge** — кожен завантажений `.riv`-файл тепер живе як нативний HybridObject один раз на view, а не пересоздається через імперативні команди старого ViewManager'а.

Цифри з власного бенчмарку авторів (iPhone 13 mini, реліз-білд):
• 24 view з одного 2.9MB файла — ~29мс замість ~2.7с, у 94 рази швидше
• 6 важких інстансів одночасно — 112MB памʼяті замість 525MB
• запис властивості — ~6.3мкс замість ~20.4мкс, у 3.2 рази швидше

У статті не пишуть, хто вже юзає нову версію в проді, і не згадують відомих багів — це свіжий реліз, ще не перевірений часом. Якщо анімації Rive і старий bridge реально гальмують апку — варто спробувати, але тримай стару версію напоготові про всяк випадок.

## 💡 Рецепти

**Reanimated/Worklets [зарелізили 0.11.0](https://github.com/software-mansion/react-native-reanimated/releases/tag/worklets-0.11.0) з experimental bytecode worklets** — перший офіційний workaround на memory-issue Hermes V1, через який Reanimated міг зʼїдати на 25-30% більше памʼяті на Android (про цю проблему ми писали ще в липневих дайджестах).

У Legacy Eval Mode тепер можна компілювати worklets у Hermes bytecode замість звичайного коду. Працює тільки для production-білдів — у Bundle Mode прапорець нічого не змінює.

```js
/** @type {import('react-native-worklets/plugin').PluginOptions} */
const workletsPluginOptions = {
  hermesBytecode: true,
  getHBCBinary: () => {
    // абсолютний шлях до бінарника hermesc для поточної машини
  },
};

module.exports = {
  plugins: [
    ['react-native-worklets/plugin', workletsPluginOptions],
  ],
};
```

Джерело — офіційна документація babel-плагіна Worklets; фіча позначена як experimental і опт-ін, тож на проді вмикай обережно і спочатку профіль на реальному білді.

Якщо вже бачив той memory-issue на Android — це перший спосіб обійти його самому, не чекаючи фікса від Meta. Якщо не бачив — можна почекати, поки фіча стабілізується.
