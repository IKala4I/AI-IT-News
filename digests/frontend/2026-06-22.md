## ⚛️ React Core

### TypeScript 7.0 RC — Go compiler, ~10× faster (June 18) ⭐ Story of the week
Microsoft shipped the RC for TypeScript 7.0, the landmark rewrite from TypeScript to Go. The VS Code codebase cold-builds in **7 seconds**, down from 77 — an 11× speedup on the exact benchmark that's been tracking TS performance for years.

**Why it matters for React devs:** Every tsc, every IDE incremental check, every CI run gets dramatically faster. Teams at Vercel, Figma, Canva, Slack, and Bloomberg participated in pre-release testing and report 8–12× wall-clock improvements in their monorepos. The API is fully type-compatible: no source changes, just a faster toolchain.

**Installation:** `npm install -D typescript@rc` + the optional *TypeScript Native Preview* VS Code extension for in-editor speed.

**Stable ETA:** ~1 month from now (mid-July 2026).

- [Announcement — devblogs.microsoft.com](https://devblogs.microsoft.com/typescript/announcing-typescript-7-0-rc/)
- [TechTimes coverage](https://www.techtimes.com/articles/318666/20260618/typescript-70-rc-ships-go-compiler-cuts-vs-code-build-time-77-seconds-seven.htm)

### React Router v8 — "intentionally boring" major release (June 17)
Brooks Lybrand released React Router v8 as the first entry in a new **yearly major release cadence** timed to Node.js EOL cycles (v8 → Node 22 EOL alignment; v9 planned ~May 2027).

**New minimums:** Node.js 22.22.0, React 19.2.7, Vite 7.0.0.

**Breaking changes:** Near-zero if you adopted all v7 future flags. The upgrade surface is almost entirely tooling (Node, Vite) rather than API.

**Why it matters:** The "boring" framing is intentional — the team is signalling that React Router will no longer be a source of surprise churn. React 19 is now a hard floor, which consolidates the React ecosystem around a single baseline.

**Migration:** Run `npx react-router@latest reveal` to see current future-flag status, then `npx react-router-codemod@latest v8` for automated transforms.

- [Remix blog post](https://remix.run/blog/react-router-v8)
- [Netlify support announcement (same day)](https://www.netlify.com/changelog/2026-06-18-react-router-8-support/)
- [GitHub discussion](https://github.com/remix-run/react-router/discussions/14468)

### Can signals make React Redux faster? — Mark Erikson (June 19)
Erikson published an experiment wiring signals-based selectors into React Redux to avoid unnecessary re-renders. Early results show notable gains in selector-heavy UIs; he frames it as a research spike rather than a roadmap item.

**Why it matters:** Signals have been a recurring theme in non-React frameworks (Solid, Vue, Angular). This is the React team's own Redux maintainer taking them seriously. Worth watching for the signal (pun noted) on where React state management is heading.

## 📱 React Native & Expo

### EAS Observe — production performance monitoring, open beta
Expo launched EAS Observe, a first-party performance monitoring service that tracks real-user startup times, rendering performance, and bundle load times across production builds. Works with OTA updates too — you can diff performance between an OTA and a native build.

**Free tier:** First 10 000 MAU free. Requires Expo SDK 55+.

**Why it matters:** React Native has historically lacked a purpose-built RUM (Real User Monitoring) solution tied to the EAS build/update pipeline. This closes the loop: build → update → observe, all in one toolchain.

- [Expo blog — Introducing Observe](https://expo.dev/blog/introducing-observe)
- [Docs](https://docs.expo.dev/eas/observe/introduction/)

## 🧰 Libraries & Tooling

### npm v12 — postinstall scripts blocked by default. July deadline. Act now.
npm v12 ships in July 2026 and is the most security-significant npm change in years: `preinstall`, `install`, and `postinstall` scripts from **dependencies** no longer run unless explicitly allow-listed in your project.

**What breaks immediately:**
- Any native addon that relies on `node-gyp` at install time (including older versions of `bcrypt`, `canvas`, `sqlite3`, `sharp`, `fsevents`, `bufferutil`)
- React Native projects with custom native modules that use `postinstall` hooks

**Background:** GitHub published breaking-change docs on June 9 — five days after the Miasma supply chain attack on June 1 (compromised 89 packages under `@redhat-cloud-services` namespace). The Microsoft Security blog on June 17 detailed the separate Mastra npm compromise, reinforcing the urgency.

**Migration now (before July):**
1. Pin npm to `≥ 11.16.0` in CI
2. Run `npm install` and read the new warnings
3. Run `npm approve-scripts --allow-scripts-pending` to generate an allowlist
4. Commit the updated `package.json`

- [GitHub Changelog — June 9](https://github.blog/changelog/2026-06-09-upcoming-breaking-changes-for-npm-v12/)
- [Microsoft Security — Mastra attack (June 17)](https://www.microsoft.com/en-us/security/blog/2026/06/17/postinstall-payload-inside-mastra-npm-supply-chain-compromise/)
- [Migration guide — npmv12guide.com](https://npmv12guide.com/)
