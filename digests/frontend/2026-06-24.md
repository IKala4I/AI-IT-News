## 📱 React Native & Expo

### Reanimated 4.5.0 — CSS pseudo-selectors, SharedTransitionBoundary, SVG on web (June 23) ⭐

The biggest library drop this week. Reanimated 4.5.0 ships three headline capabilities:

**1. CSS animation pseudo-selectors with native backends**
`:hover`, `:active`, and `:focus` now work across **web, Android, and Apple** platforms. Styles declared in `useAnimatedStyle` or the CSS animation API respond to interaction states with native-tier performance (no JS bridge round-trip for the Apple and Android paths).

**2. `<SharedTransitionBoundary>` component**
Scopes shared element transitions to a subtree. Previously, all matching elements on both screens participated in a shared transition; the new boundary lets you limit scope to a specific part of the screen, preventing unwanted cross-screen animation interference.

**3. CSS animations for `react-native-svg` on web**
SVG elements from `react-native-svg` can now be animated via the Reanimated CSS animation API on web targets, closing a long-standing gap for teams that share animation code across RN and web.

**Also in this release:**
- `shadowColor`, `shadowOpacity`, `shadowRadius`, `shadowOffset`, `background`, and `border` CSS transitions now run on **iOS Core Animation** (enable via `IOS_CSS_CORE_ANIMATION` feature flag)
- Fixed 30fps layout animation regression on iOS release builds
- Fixed exiting views getting stuck under heavy load on Android
- Fixed memory leaks when unmounting animated views
- Fixed Android ANR deadlocks in CSS update registry

**Breaking changes:** None documented. The `IOS_CSS_CORE_ANIMATION` flag is opt-in.

**Worklets 0.10.0** (June 19, covered in the June 22 digest) is a companion release; upgrade both together.

- [GitHub release — Reanimated 4.5.0](https://github.com/software-mansion/react-native-reanimated/releases/tag/4.5.0)

### Expo Maestro Testing — insights dashboard (June 24)

Expo added a new insights dashboard to EAS Maestro e2e testing. The dashboard provides:
- **Flakey test identification** across runs
- **Performance tracking** per test suite
- **Regression detection** to catch suite slowdowns before they hit production

**Why it matters:** Flakiness is the #1 reason teams abandon e2e testing in RN. Surfacing it at the CI level (rather than in dev Slack complaints) gives teams data to triage and fix unstable tests before they erode CI trust. Pairs with the EAS Observe RUM tooling announced last week.

## ▲ Next.js / Vercel

### Vercel Flags — platform-native feature flags (June 22) ⭐

Vercel shipped a first-party feature flag solution, **Vercel Flags**, embedded directly in the Vercel platform. Key design choices:

- **Server-side defaults:** Flags are evaluated at the edge/server before the response is sent, so there's no client-side flicker or layout shift from flag resolution.
- **"Zero impact on page performance":** Because evaluation is server-side, the flag SDK adds no client bundle weight and no waterfall request.
- **No credential storage:** Uses Vercel's existing short-lived token model (similar to Vercel Connect) rather than a separate API key you manage.

**Why it matters for Next.js devs:** Third-party flag providers (LaunchDarkly, Statsig, Unleash) typically require a client SDK that adds JS weight and a round-trip before a flag resolves. Vercel Flags sidesteps both by running the evaluation in your Next.js Server Component or middleware — you branch on the flag value just like any server variable.

**Integration:** Still in public beta; docs available at `vercel.com/docs/flags`.

- [Vercel Blog — Vercel Flags](https://vercel.com/blog)

### Next.js canary.61–.65 roundup (June 22–24)

All pre-release only; stable track remains **16.2.4** (April 15). Notable signal for what's coming in 16.3:

| Canary | Date | What to know |
|---|---|---|
| .61 | June 23 | **Legacy PPR codepaths removed** — the experimental Partial Pre-Rendering toggle no longer has a fallback path; Cache Components approach is now the sole model. React internals updated to `247fbb45-20260622`. |
| .62 | June 23 | False-positive fix for `export const dynamic` detection in Cache Components; internal skill cleanup. |
| .63 | June 24 | **ISR fallback shells now served on prefetch requests** — prefetch hits can now receive the ISR shell rather than a full uncached response, improving perceived navigation speed. Navigation inspector UI updated. |
| .64 | June 24 | Turbopack gains **single-entry chunk support**; chunk-merging heuristic improved (`overlap == 1` case); `TURBOPACK_DEBUG_CSS_CHUNKING` env var added for debugging CSS splits. |
| .65 | June 24 | Cache lifetime now preserved for `cacheMaxMemorySize: 0` caches in dev; internal request header mutation bug fixed. |

**Take-away:** The 16.3 canary stream is converging on Cache Components as the stable PPR replacement, with Turbopack chunk-splitting improvements landing in parallel. Expect a stable release proposal in the coming weeks.
