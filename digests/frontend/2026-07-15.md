## 📱 React Native та Expo

**Callstack розібрали, як React Navigation 8 [прибирає боілерплейт із типізації навігації](https://www.callstack.com/blog/less-boilerplate-more-type-safety-react-navigation-8s-typed-hooks)** — `useRoute`, `useNavigation` і `useNavigationState` тепер приймають назву екрана і повертають точний тип без generics і `as`.

```ts
const route = useRoute('Profile'); // route.params типізовано під Profile
```

Хук перевіряє під час рантайму, що ти дійсно всередині потрібного екрана, тож типи й реальність більше не розходяться — раніше для цього доводилось руками писати generic-параметр або кастити через `as`.

Фіча вже в React Navigation 8, яка сама ще в альфі (`8.0.0-alpha.33`) — стабільної версії поки нема, ставити варто через `next`-тег і без production-апки поруч.

Якщо вже пробуєш React Navigation 8 — це відчутно зменшить boilerplate у навігаційному коді. Якщо сидиш на стабільній 7.x — просто тримай в голові на майбутнє.

#reactnative

**Expo [показали на практиці](https://expo.dev/blog/nesting-tabs-and-drawers-with-expo), як акуратно вкласти Tabs всередину Drawer в Expo Router** — коли апці потрібне і бічне меню, і нижні таби, а жести не повинні заважати одне одному.

Пост розбирає структуру `app/`-директорії для такого layout і нюанси з темізацією та gesture handling, щоб свайп drawer не з'їдав свайпи табів.

Якщо в тебе саме такий кейс навігації — готовий референс, а не теорія з нуля.

## 🧰 Бібліотеки й тулінг

**Vite [зарелізили 8.1.4](https://github.com/vitejs/vite/releases/tag/v8.1.4)** — переважно фікси, але є одна фіча під прапорцем: новий `oxc`-мінімізатор як альтернатива дефолтному.

Що ще підвезли:
• чанки з чистим CSS тепер вирізаються з imports, коли увімкнено `chunkImportMap`
• коректний `import.meta.url` у preload-функціях
• вирівняні SSR stack-trace колонки під Node
• апдейт `postcss-modules` до v9

Дрібний патч, але якщо ти на 8.1.x — оновлюватись безболісно, ламати нічого не повинно.
