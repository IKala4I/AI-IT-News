### EU Names EUROPA Consortium Winner of Frontier AI Grand Challenge

On June 19, the European Commission named the EUROPA consortium — led by Italian AI firm Domyn — as the winner of its Frontier AI Grand Challenge. The project will build a frontier open-source AI model exceeding 400 billion parameters, natively supporting all 24 EU official languages, trained and run on European soil under EU governance. Infrastructure support includes access to up to 2.5% of EuroHPC supercomputing capacity for one year on dedicated AI-optimized clusters. When complete, the model weights will be made broadly available to EU public authorities, scientific institutions, and businesses.

**Why it matters:** Europe has historically watched the US and China race ahead on frontier AI; the EUROPA project is the most concrete EU attempt to establish sovereign AI infrastructure that doesn't depend on US export control exemptions. The timing — announced a week into the Fable/Mythos shutdown — makes the strategic motivation explicit: European institutions experienced in real time what US supply-chain dependency means when a US directive removes model access with no warning. A 400B+ open model trained under EU law is the structural answer.

### OpenAI and Broadcom Unveil "Jalapeño" — First Custom LLM Inference Chip (Breaking)

OpenAI and Broadcom announced "Jalapeño" on June 24: OpenAI's first custom AI inference chip, co-designed from scratch with Broadcom and manufactured in just nine months — one of the fastest ASIC development cycles ever achieved for advanced semiconductors. The chip is architected specifically around LLM inference workloads: optimized for the memory movement, networking, and serving patterns that matter most in frontier model serving. OpenAI says early testing shows "substantially better performance per watt" than current state-of-the-art. Initial commercial deployments are targeted for late 2026, scaling toward gigawatt-scale data centers with Microsoft and other partners. OpenAI's own models were used to accelerate parts of the chip design and optimization process.

**Why it matters:** Custom silicon is how Google (TPUs), Meta (MTIA), and Amazon (Trainium/Inferentia) all achieved meaningful cost advantages in AI inference. NVIDIA currently supplies virtually all of OpenAI's inference compute at a margin that directly affects OpenAI's unit economics. Jalapeño is OpenAI's first concrete step toward reducing that dependency — and toward owning more of the full-stack economics of serving its own models. A nine-month tape-to-chip cycle, if it holds up commercially, also hints at how AI-assisted chip design is changing hardware development timelines.

## 💻 IT / General Tech News

### "Cordyceps": A Structural CI/CD Flaw Exposes Microsoft, Google, Apache, and Cloudflare

Security firm Novee disclosed a systemic class of GitHub Actions vulnerabilities it named "Cordyceps," after the parasitic fungus that takes over its host. The flaw is structural: a low-privilege workflow triggered by an untrusted pull request feeds output into a high-privilege workflow, which then authenticates to cloud infrastructure with the highest available permissions. Every individual step looks normal; the combination creates a clear path to full repository and cloud credential compromise — exploitable by any unauthenticated user with a free GitHub account. Novee confirmed over 300 repositories were fully exploitable, including repositories owned by Microsoft, Google, Apache, Python Software Foundation, and Cloudflare. Critically, AI coding agents are spreading the flaw: they reproduce the same insecure CI/CD configuration patterns at scale from training data, quietly planting the vulnerability across new repositories as teams adopt AI-generated workflow files.

**Why it matters:** Unlike a targeted exploit, Cordyceps is a structural architectural flaw that exists wherever a specific pattern of GitHub Actions configuration appears — and AI coding tools are copying that pattern into millions of new repositories. The combination of wide attack surface, no-privilege entry requirement, and AI amplification makes this the most systemic CI/CD supply chain risk disclosed in years. Security teams should audit all GitHub Actions workflows for the untrusted-PR-to-high-privilege-workflow chain immediately — and treat AI-generated CI/CD configuration as untrusted by default until verified.

### Cursor "Origin": A Git Forge Designed for AI Agents, Not Humans

Cursor announced "Origin," a GitHub-alternative git hosting platform, on June 17. Unlike GitHub, GitLab, and Bitbucket — all designed for human-paced development — Origin treats AI agents as first-class citizens. The platform is built around the premise that a significant portion of code is already written by autonomous agents: its headline demo showed 22.6 commits per second, 296,000 clones per hour, and sub-400-millisecond global sync latency. It automatically handles merge conflicts generated by parallel agent work. Origin is built on Graphite (acquired by Cursor/Anysphere in December 2025). General availability is targeted for fall 2026; a waitlist is open now. No public pricing has been announced.

**Why it matters:** GitHub's permission model, review workflow, and merge infrastructure were designed for humans doing code review at human speed. Once a significant fraction of commits are coming from hundreds of concurrent agents, the fundamental assumptions of that design break. Origin is a bet that the code collaboration layer needs to be rebuilt from scratch for the agentic era — rather than patched incrementally. With 50%+ Fortune 500 install base on Cursor IDE (and the SpaceX acquisition closing in Q3), Anysphere has the distribution to make Origin the default infrastructure for AI-first development teams.
