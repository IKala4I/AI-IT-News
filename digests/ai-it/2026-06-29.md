### Amazon Q Developer Vulnerability Let Malicious Repos Steal AWS Credentials (CVE-2026-12957/12958)

Security researcher Maor Dokhanian of Wiz Research disclosed two now-patched vulnerabilities in Amazon Q Developer (the AI coding assistant for VS Code) on June 26 under Security Bulletin 2026-047-AWS. The root cause: Amazon Q automatically loaded MCP server configurations from `.amazonq/mcp.json` files in any opened workspace without user consent or workspace trust verification. A single malicious config file in a repo — no prompts, no clicks — could exfiltrate active AWS session credentials to an attacker-controlled server the moment a developer opened and trusted the workspace. A proof-of-concept was demonstrated by Wiz. Amazon deployed an initial fix on May 12 and full public disclosure followed June 26. The patch ships in Language Servers for AWS v1.69.0, which auto-updates for most IDE users; reloading the IDE triggers the update.

**Why it matters:** Opening a Git repository is one of the most routine developer actions. An attack that triggers at "open repo" — with no further interaction — and immediately harvests cloud credentials is about as low-friction as exploits get. For teams using Amazon Q with high-privilege AWS roles (common in DevOps and platform engineering), a malicious repo in a PR could translate directly to account compromise. The MCP auto-load pattern is not unique to Amazon Q; other AI coding tools with MCP integration may have similar assumptions worth auditing.

### pedit COW (CVE-2026-46331): Linux Kernel Priv-Escalation Exploit Is Public — Patch Now

A public, author-verified proof-of-concept exploit nicknamed "packet_edit_meme" appeared June 17, allowing any local unprivileged Linux user to obtain a root shell on systems where the `act_pedit` traffic-control action is reachable and unprivileged user namespaces are available. The underlying bug is in `tcf_pedit_act()`: it computes its copy-on-write range once before editing packet headers, misses the runtime offset added by typed keys at execution time, and writes into a shared page-cache page it never made private. The exploit poisons an in-memory copy of a system binary, making the compromise invisible to file-integrity monitoring. Red Hat issued RHSB-2026-008. Mitigation without patching: block the `act_pedit` module to prevent it from being auto-loaded at boot. Full kernel patches are shipping from major distributions.

**Why it matters:** A working, published root-privilege exploit for Linux is an immediate operational risk for every Linux-based server, container host, and developer machine where unprivileged user namespaces are enabled (the default in most distributions). The fact that the compromise is invisible to file-integrity monitoring makes it particularly dangerous in environments relying on audit tooling for intrusion detection. Apply distribution patches or the module-block mitigation now; don't wait for the next maintenance window.

### Fugu Ultra (Sakana AI): A Japanese LLM That Orchestrates Other Models as Subtasks

Sakana AI released Fugu Ultra, a Japanese language model that approaches Western flagship model performance on Japanese benchmarks via a novel architecture: rather than scaling a single model, Fugu Ultra generates natural-language workflows, decomposes tasks into subtasks, and recursively distributes those subtasks across a panel of smaller specialist models. The result is a system that can handle complex multi-step tasks by routing through a coordinated network rather than a monolithic forward pass.

**Why it matters:** Fugu Ultra is a practical production implementation of the "LLM as orchestrator" pattern that researchers have been exploring for the past two years. If the approach generalizes — using workflow generation and model routing to approximate frontier-model capability without frontier-model training costs — it represents a meaningful architectural alternative to parameter scaling. For non-English languages where frontier training data is thinner, orchestration-based approaches may prove especially practical.
