## 🛠️ AI Tools & Releases

### Claude Code v2.1.208 and v2.1.209 shipped

**What happened:** Anthropic published two Claude Code releases on July 13–14. v2.1.208 adds a screen-reader mode (`claude --ax-screen-reader`, or `CLAUDE_AX_SCREEN_READER=1`) that renders plain text for accessibility tools. It also adds `vimInsertModeRemaps` (map a two-key sequence like `jj` to Escape in vim mode), a `CLAUDE_CODE_PROCESS_WRAPPER` env var so corporate launchers can force every spawn through a required wrapper, and mouse-click support in fullscreen multi-select menus. The release fixes several bugs, including the context window falsely resetting to 200k after an auto-update, background sessions crashing on an HTTP/2 GOAWAY, and `CLAUDE_CODE_MAX_OUTPUT_TOKENS` mishandling scientific notation (`1e6` became `1`). v2.1.209, the same day, is a small patch that reverts an over-broad guard that had blocked `/model` and other dialogs inside background `claude agents` sessions.

**Why it matters to me:** Direct upgrades to my daily coding tool. The context-reset and max-output-token fixes remove two silent foot-guns, and the vim remap plus screen-reader mode are quality-of-life wins.

Source: [Claude Code CHANGELOG](https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md) · [Docs changelog](https://code.claude.com/docs/en/changelog)

### GitHub Copilot: `/security-review` now runs inside the Copilot app

**What happened:** GitHub moved its `/security-review` slash command into the GitHub Copilot app as a public preview (previously it lived in the CLI/IDE). You run it against your current changes and it returns findings scored by severity and confidence for common vulnerability classes — injection, cross-site scripting (XSS), insecure data handling, path traversal, and weak crypto — with suggested fixes. It complements code scanning, Dependabot, and secret scanning, and is available to all Copilot tiers (Free, Pro, Business, Enterprise) during the preview.

**Why it matters to me:** A quick, in-app self-check for security bugs before I open a pull request — one more automated reviewer that costs nothing to try during the preview.

Source: [GitHub Changelog (July 14)](https://github.blog/changelog/2026-07-14-security-reviews-now-available-in-the-github-copilot-app/)

### DOOMQL: a Doom-style game built entirely in SQL

**What happened:** Simon Willison wrote up DOOMQL, a first-person Doom-like game by Peter Gostev where SQLite does everything — movement, collision, enemies, combat, and even pixel rendering — through recursive CTEs (a SQL feature where a query calls itself). A thin Python host only passes keyboard input and prints the query's output; there are no third-party Python dependencies, and it ships with 36 tests. Gostev reportedly built the SQL/game logic with GPT-5.6 Sol. Simon then used Claude (Fable 5) to build a companion view on top of Datasette that shows the live game state and a SQL-driven minimap.

**Why it matters to me:** A fun, concrete demo of how far pure SQL can be pushed, and a practical hands-on example of pairing two AI coding models on one project (one writes the engine, one builds the tooling around it).

Source: [simonwillison.net](https://simonwillison.net/2026/Jul/13/doomql/) · Repo: [petergpt/doomql](https://github.com/petergpt/doomql)

### `destructive_command_guard`: a safety hook for coding agents

**What happened:** A trending open-source Rust tool, `destructive_command_guard`, hooks into Claude Code, Codex CLI, GitHub Copilot CLI, and Cursor as a `PreToolUse` guard. It intercepts and blocks dangerous commands — like `git reset --hard`, `rm -rf ./src`, or `DROP TABLE users` — before an AI agent runs them. It ships 50+ pattern packs (databases, Kubernetes, cloud providers), is MIT-licensed, and installs from a single curl script. The repo was gaining hundreds of stars a day when I checked.

**Why it matters to me:** I run agents that execute shell commands, sometimes unattended. A local guardrail that refuses the truly destructive ones is cheap insurance against a bad tool call wiping real work.

Source: [github.com/Dicklesworthstone/destructive_command_guard](https://github.com/Dicklesworthstone/destructive_command_guard)

## 💻 IT & Dev News

### npm v12 will block install scripts by default — this month

**What happened:** npm v12, described as npm's biggest security redesign in 16 years, ships this month. `npm install` will no longer auto-run `preinstall`/`install`/`postinstall` scripts or native builds (like `node-gyp rebuild`) by default. Custom Git-URL dependencies are blocked by default, and packages pulled from arbitrary remote URLs or tarballs outside official registries are disallowed unless you approve them. You can upgrade to npm 11.16.0+ now to get warnings early and use the new `npm approve-scripts` command to build an allowlist before the breaking change lands.

**Why it matters to me:** This directly changes how `npm install` behaves and can break builds that rely on postinstall scripts or git dependencies. Testing my projects against npm 11.16.0+ now — and pre-approving the scripts I actually need — avoids a surprise when v12 arrives.

Sources: [Infosecurity Magazine](https://www.infosecurity-magazine.com/news/github-update-npm-supply-chain/) · [Tech Times](https://www.techtimes.com/articles/319890/20260708/npm-v12-ships-this-month-blocking-install-scripts-that-enabled-year-supply-chain-attacks.htm)

### 148 malicious "student proxy" npm packages — still not fully cleaned up

**What happened:** The Hacker News published an updated account of a long-running npm campaign. Packages disguised as "student proxy" tools (for bypassing school web filters) hid a remote script loader and a WebSocket-flood module that turned each visiting browser into part of a DDoS botnet. The campaign started as adware in March, added the malicious modules in mid-May, and relaunched on July 8 under a new publisher account, reaching 148 packages in total. As of a July 14 check, most package families were pulled and replaced with npm's `0.0.1-security` placeholder — but the "charlie-kirk" family was still live, serving the exact versions researchers flagged as malicious. Because the loader points at a mutable GitHub branch (through the jsDelivr CDN) instead of a pinned commit, the operator can re-arm the payload with a single commit, without republishing the package.

**Why it matters to me:** A reminder that "the package was taken down" does not always mean "the threat is gone." Pinning dependencies and watching for loaders that fetch code from mutable sources is the practical defense.

Source: [The Hacker News](https://thehackernews.com/2026/07/148-npm-packages-disguised-as-student.html)

### Microsoft Patch Tuesday: 127 CVEs, plus a Defender privilege-escalation bug

**What happened:** Microsoft's July 14 Patch Tuesday fixed 127 CVEs across Windows, Office, SharePoint Server, Visual Studio, and .NET. The headline fix is **RoguePlanet (CVE-2026-50656)**, a race-condition flaw in the Microsoft Malware Protection Engine (`mpengine.dll`, the Defender scanning engine) that let a standard user reach `NT AUTHORITY\SYSTEM` through improper link resolution. Microsoft actually shipped the engine fix out-of-band on July 8 after a public proof-of-concept appeared; Defender auto-updates the engine, so no manual action is needed for that part. This month's update also completes the final phase of Kerberos RC4 enforcement, which disables the old RC4 cipher in Kerberos authentication by default and can break legacy systems that still rely on it.

**Why it matters to me:** The Visual Studio and .NET fixes are on my own dev machine, and the Kerberos RC4 change can break authentication in older internal tools — worth checking before it bites. The Defender fix is automatic, which is good news for once.

Sources: [The Hacker News](https://thehackernews.com/2026/07/microsoft-patches-rogueplanet-defender.html) · [Help Net Security](https://www.helpnetsecurity.com/2026/07/09/microsoft-releases-fix-for-rogueplanet-defender-flaw-cve-2026-50656/)

### SCALE: run CUDA code on AMD GPUs without rewriting it

**What happened:** Spectral Compute's SCALE toolchain resurfaced as a top Hacker News discussion this week. SCALE is a clean-room, CUDA-compatible toolchain — a drop-in replacement for `nvcc` plus a runtime that impersonates the CUDA Toolkit — that lets existing CUDA codebases compile and run natively on AMD RDNA2/RDNA3 GPUs with no code changes. It is proprietary (there is a free edition, but it is not open source), has been in development for about seven years, and has reportedly run on the Frontier exascale supercomputer.

**Why it matters to me:** If it works as described, it lowers the lock-in of CUDA, letting GPU code target AMD hardware without a rewrite. That is worth a look for anyone doing GPU compute who wants a hardware fallback.

Source: [HPCwire](https://www.hpcwire.com/2026/07/09/spectral-compute-aims-to-set-cuda-free-will-it-succeed/)

## 🔁 Follow-ups

### Grok Build CLI: xAI admits the codebase upload, ships a server-side fix

**What happened:** Yesterday's digest covered a researcher's finding that xAI's Grok Build CLI silently uploaded entire git repositories — including `.env` secrets — to xAI's cloud on every session. xAI has now acknowledged the issue. According to The Register (July 14), Elon Musk responded the same day and said the uploaded data "will be completely and utterly deleted," and xAI shipped a server-side flag (`disable_codebase_upload: true`) that stops new uploads. The original researcher (Cereblab) confirmed uploads have stopped but criticized the tool's `/privacy` command as only a per-session retention toggle, not a full fix.

**What changed / still open:** New uploads are stopped and the issue is acknowledged. But there is no independent confirmation that already-collected data was deleted, no timeline, and no count of affected users.

Sources: [The Register](https://www.theregister.com/ai-and-ml/2026/07/14/musk-promises-purge-after-grok-build-caught-sending-entire-repos-to-the-cloud/5271123) · [Original disclosure (gist)](https://gist.github.com/cereblab/dc9a40bc26120f4540e4e09b75ffb547)

### GhostLock (CVE-2026-43499): patches landing unevenly across distros

**What happened:** The Linux kernel "GhostLock" flaw (a futex use-after-free that gives near-certain local root escalation) is now getting fixes, but the rollout is uneven. AlmaLinux shipped patched kernels for versions 8, 9, and 10 on July 9. CloudLinux has patches rolling from testing to stable, with a KernelCare livepatch already live for CloudLinux 9. Red Hat's security bulletin (RHSB-2026-010) still says it is "investigating mitigation options" — no RHEL patch yet. Standard Ubuntu LTS kernels also appear not fully patched as of this check.

**What changed / still open:** RHEL-family rebuilds (AlmaLinux, CloudLinux) are furthest along; upstream RHEL and Ubuntu LTS are still pending. No source confirms the rollout is complete anywhere. If you run Linux servers, check your distro's advisory and patch as fixes land.

Sources: [AlmaLinux advisory](https://almalinux.org/blog/2026-07-09-ghostlock/) · [Red Hat RHSB-2026-010](https://access.redhat.com/security/vulnerabilities/RHSB-2026-010)
