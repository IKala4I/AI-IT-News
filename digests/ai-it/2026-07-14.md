## 🚀 Моделі

**PrismML [випустили Bonsai 27B](https://prismml.com/news/bonsai-27b) — 27B-модель, яка реально влазить у телефон**

Ternary-версія (5.9 GB) тримає 95% якості full-precision бейзлайна, а 1-bit версія (3.9 GB) — 90%, і саме вона запускається на iPhone 17 Pro. Модель мультимодальна (vision), контекст 262K токенів, є tool calling і speculative decoding.

Ліцензія Apache 2.0, ваги викладені на Hugging Face, є безкоштовне preview API і демо на HF Spaces.

Якщо давно хотілось локальну модель з нормальним рівнем інтелекту прямо на телефоні, а не маркетинговий тизер, — це перший реальний кандидат.

## 💡 Рецепти

**Рецепт: [підміни, що каже Claude Code](https://jola.dev/posts/how-to-stop-claude-from-saying-load-bearing) — хук `MessageDisplay`, який перехоплює вивід на льоту**

Claude Code вміє перехоплювати текст прямо перед виводом через хук `MessageDisplay`. Конфіг у `~/.claude/settings.json`:

```json
{
  "hooks": {
    "MessageDisplay": [
      { "hooks": [ { "type": "command", "command": "$HOME/.claude/hooks/wordswap.sh" } ] }
    ]
  }
}
```

А сам хук (`~/.claude/hooks/wordswap.sh`, виконуваний) читає stdin як JSON і повертає підмінений текст:

```python
#!/usr/bin/env python3
import json, re, sys
replacements = {"load-bearing": "cooked", "seam": "whatchamacallit"}
data = json.load(sys.stdin)
text = data.get("delta") or ""
for phrase, repl in replacements.items():
    text = re.sub(r"\b" + re.escape(phrase) + r"\b", repl, text, flags=re.IGNORECASE)
print(json.dumps({"hookSpecificOutput": {"hookEventName": "MessageDisplay", "displayContent": text}}))
```

Хуки вантажаться при старті, тож досить нової сесії. Сам не тестив, але механізм реальний і задокументований автором у пості.

Жарт із заміною фраз — привід, а насправді це наочний доказ, що `MessageDisplay` hook існує і може переписувати вивід агента на льоту — придатно і для серйозніших речей типу маскування секретів у виводі.

## 📌 Рухи й безпека

**xAI [зупинила зливання цілих репозиторіїв](https://www.theregister.com/ai-and-ml/2026/07/14/musk-promises-purge-after-grok-build-caught-sending-entire-repos-to-the-cloud/5271123) з Grok Build CLI — але видалення вже завантаженого не підтверджено**

Дослідник з Cereblab виявив, що Grok Build (CLI-тул xAI) вантажив у Google Cloud Storage цілі репозиторії разом з git-історією — включно з уже видаленими секретами. В одного юзера так пішла ціла директорія з SSH-ключами і базою пароль-менеджера.

xAI вимкнула це на сервері прапорцем `disable_codebase_upload`, а Маск публічно пообіцяв видалити вже завантажене — але незалежного підтвердження чи дедлайну немає. Команда `/privacy`, яку xAI рекламувала як контроль над цим, насправді лише per-session retention-тумблер, а не те, що реально зупинило вивантаження.

Якщо колись ставили Grok Build CLI — час поротейтити секрети; обіцянка видалення поки що тільки на словах.
