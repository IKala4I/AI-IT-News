# AI & IT Digest — 2026-07-14

## ⚡ TL;DR
- Claude Code shipped two quick updates (v2.1.208 + v2.1.209): screen-reader mode, a vim remap, and fixes for two silent bugs.
- GitHub Copilot's `/security-review` now lives inside the Copilot app — free security scans before you push.
- Two new agent tools worth a look: Juggler (a visual GUI for coding agents) and destructive_command_guard (blocks catastrophic agent commands).

## 🚀 Models

### OpenAI [eases GPT-5.6 usage limits](https://winbuzzer.com/2026/07/14/openai-eases-gpt-56-timing-limits-keeps-weekly-caps-xcxwbn/)

OpenAI removed the 5-hour usage window for Codex and ChatGPT Work (Plus, Business, Pro) after usage doubled post-launch. Weekly caps stay. Every account got a one-time banked reset; a context-window bug (272K instead of 372K tokens) is being fixed.

**Why:** More breathing room on Codex before hitting a wall, if you've been rationing sessions.

## 🛠️ Tools

### Claude Code [ships v2.1.208 and v2.1.209](https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md)

Anthropic shipped two Claude Code releases: a screen-reader mode, a vim remap for `jj`→Escape, and fixes for a fake context-window reset and a scientific-notation bug in token-limit settings. A same-day patch then undid an over-broad guard blocking dialogs in background sessions.

**Why:** Two silent foot-guns are fixed — worth the update today.

### GitHub Copilot's [`/security-review` moves into the app](https://github.blog/changelog/2026-07-14-security-reviews-now-available-in-the-github-copilot-app)

GitHub moved `/security-review` out of the CLI/IDE and into the Copilot app, as a public preview. It scans your current changes for common bugs — injection, XSS, path traversal, weak crypto — with suggested fixes, free on every Copilot tier.

**Why:** A free in-app check for security bugs before you open a PR.

### Juggler [gives coding agents a visual, branching GUI](https://github.com/juggler-ai/juggler)

A new desktop/browser app replaces the linear chat transcript with an editable, branching session tree and a Miller-column view of every tool call. Works with Claude Code, Codex, Gemini, Ollama, and OpenRouter.

**Why:** Real visibility into what your coding agent is doing, and a way to steer it.

### `destructive_command_guard` [stops agents from running catastrophic commands](https://github.com/Dicklesworthstone/destructive_command_guard)

A free, open-source hook for Claude Code, Cursor, Copilot CLI, Codex, and more. It intercepts and blocks commands like `git reset --hard` or `rm -rf` before your agent runs them. One-line curl install.

**Why:** Cheap insurance if you run agents unattended — a wrong tool call can't wipe your work.

## 📌 Moves & Security

### xAI [fixes the Grok Build CLI's data leak](https://www.theregister.com/ai-and-ml/2026/07/14/musk-promises-purge-after-grok-build-caught-sending-entire-repos-to-the-cloud/5271123) — but deletion is still unverified

Follow-up: xAI's Grok Build CLI (yesterday's story) stopped uploading whole repos after `disable_codebase_upload` flipped server-side. Musk publicly promised the already-uploaded data — including `.env` secrets — will be fully deleted, but there's still no independent proof or timeline.

**Why:** Rotate any secrets it touched — the deletion promise is unverified.
