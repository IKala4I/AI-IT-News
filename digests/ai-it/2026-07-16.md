## 🚀 Моделі

**Thinking Machines Lab (лабораторія Міри Мураті) [випустила Inkling](https://thinkingmachines.ai/news/introducing-inkling/) — 975B-модель з відкритими вагами, найбільшу з розроблених у США**

MoE-архітектура: 975B параметрів, з яких активних лише 41B, контекст до 1M токенів. Модель мультимодальна — текст, зображення, аудіо, відео, тренована на 45 трильйонах токенів. Є легша Inkling-Small (276B параметрів, 12B активних) для дешевшого й швидшого інференсу.

Ліцензія Apache 2.0, ваги вже на Hugging Face, файнтюнити можна через Tinker (тимчасова знижка 50%), а інференс одразу підключили TogetherAI, Fireworks, Modal, Databricks і Baseten.

Позиціонують не як найрозумнішу модель на ринку, а як збалансований опенсорсний фундамент під кастомізацію під конкретну задачу — цікаво, чи так і виявиться на практиці.

## 🛠️ Інструменти

**Claude Code [оновився до 2.1.211](https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md) — захист від спуфінгу в permission-запитах і чесніші background-агенти**

Найважливіше: permission-превью тепер нейтралізують bidirectional-override, zero-width і look-alike символи — раніше ними можна було візуально підмінити текст запиту на дозвіл дії так, щоб людина побачила не те, що насправді підтверджує. Background-агенти більше не репортять фейкове "завершено", якщо насправді впали.

Ще з нового: прапорець `--forward-subagent-text`, щоб бачити текст і думки субагентів у stream-json, підтримка scientific notation (`1e6`) у лімітах токенів і купа дрібних UI-фіксів.

Дрібниці, які по одній непомітні, але саме вони визначають, чи можна довіряти тулу, коли він працює у фоні без нагляду.

## 💡 Рецепти

**Рецепт: [autoloop](https://github.com/yaoshengzhe/autoloop) — пайплайн, який сам тріажить беклог, пише код і відкриває PR за $110/місяць**

Автор [описав](https://andywidjaja.com/blog/110-pipeline) власний сетап: один конфіг-файл і три systemd-таймери (тріаж о 00:00 UTC, імплементація о 02:00, changelog о 04:00) — без черг і оркестраторів.

```toml
# autoloop.toml
repo = "your-org/your-repo"
triage_model = "sonnet"
impl_model = "claude-opus-4-6"
verify_cmd = "uv run pytest"
max_retries = 3
protected_paths = ["autoloop/", "autoloop.toml"]
```

```bash
claude plugin marketplace add yaoshengzhe/autoloop
claude plugin install autoloop@autoloop
autoloop init --repo your-org/your-repo --verify-cmd "pytest"
```

Ціна за словами автора: $100/міс за підписку Claude Max 5x + $9.99 за VPS на 2 vCPU. Файли з `protected_paths` агент змінювати не може — сам собі апгрейд конфігу не влаштує.

Автор ділиться власним, реально працюючим сетапом з цифрами — це не переказ чужого допису, але й не перевірено незалежно кимось третім.

## 📌 Рухи й безпека

**xAI [опублікувала повний код Grok Build](https://github.com/xai-org/grok-build) на GitHub — 844 тисячі рядків Rust під Apache 2.0**

Продовження вчорашньої історії: CLI-тул xAI виявили за тихим вивантаженням цілих репозиторіїв (разом з `.env` і SSH-ключами) в хмару. Тепер, вимкнувши цю функцію на сервері, xAI викотила весь агентський код у відкритий доступ — agent loop, інструменти, TUI, систему плагінів і MCP.

Контриб'юції ззовні не приймають (див. CONTRIBUTING.md) — це "форкни й запускай", а не "спільно розвивай". Довіру одним опенсорсом миттєво не купиш, але можливість самому прочитати код агента, яким користуєшся, — це вже більше, ніж було тиждень тому.
