### Anthropic Research: Domain Expertise Beats Coding Background in Agentic Work

Anthropic published "Agentic coding and persistent returns to expertise," analyzing ~400,000 Claude Code sessions from October 2025 through April 2026. The headline finding: domain knowledge matters more than programming background for productive agentic sessions. Non-software-engineers achieved a 29% verified success rate versus software engineers' 34% — within 5 points. Expert users across all fields outperformed novices by a wide margin regardless of occupation. Users directed ~70% of planning decisions (what to build, what counts as done); Claude handled ~80% of execution decisions (which files to change, which commands to run). Estimated session value rose 27% over the period; the share of sessions spent fixing broken code fell from 33% to 19%.

**Why it matters:** AI coding tools democratize code production for domain experts — doctors, scientists, lawyers — in a real, measurable way. The near-parity between software engineers and non-engineers in success rates is the clearest evidence yet that the constraint on productive agentic work is understanding the problem, not knowing Python. The session quality improvement over time also suggests the tools are getting genuinely better at execution, not just accumulating more capable users.

## 💻 IT / General Tech News

### "Agentjacking": Fake Bug Reports Hijack AI Coding Agents

Security firm Tenet disclosed a novel attack targeting AI coding agents (Claude Code, Cursor, and others). Attackers identify a target's public Sentry Data Source Name (DSN), then submit a malicious error report with Markdown injection that mimics legitimate Sentry remediation guidance. When an AI agent investigates via Sentry's MCP server integration, the injected text causes it to execute attacker-controlled shell commands with the developer's local permissions. Tenet's proof-of-concept got agents to download and run a malicious npm package. Over 2,388 organizations with exposed Sentry DSNs were identified as vulnerable; AI assistants at more than 100 companies executed Tenet's test payload, including at least one Fortune 100 firm. Sentry added a content filter after Tenet's June 3 disclosure, but a deep fix is architecturally difficult: the root problem is agents treating untrusted external data as trusted instructions.

**Why it matters:** Prompt injection through agentic tool integrations is no longer theoretical — it has been demonstrated at scale on production infrastructure. The Sentry MCP vector is particularly dangerous because error reports are high-trust: developers expect them to contain actionable guidance. As AI coding agents gain deeper filesystem and shell access, a single compromised tool integration can translate directly to remote code execution. Expect MCP integration security auditing to become a required step in enterprise AI tool procurement.

### SubQ 1.1 Small: 12 Million Token Context Window at 64× Less Compute

Miami startup Subquadratic released SubQ 1.1 Small, a language model using a novel "Subquadratic Sparse Attention" (SSA) architecture that scales linearly with context length instead of quadratically. Published numbers: 12 million token context window; 64.5× less compute than dense attention at 1M tokens; 56× faster than FlashAttention-2 at equivalent context; 99.12% on RULER at 128K tokens; 85.4% GPQA Diamond; 89.7% LiveCodeBench pass@4. Currently deploying to select design partners, with a broader 2M–12M token lineup planned later in 2026. VentureBeat reports that independent researchers are demanding external verification of the claimed 1,000× efficiency figures.

**Why it matters:** Quadratic attention scaling is a hard architectural constraint: every doubling of context costs 4× the compute, which is why even well-funded labs cap context windows. Linear scaling, if it holds up, would make 12M-token contexts economically practical — enough to fit most codebases, multi-year email histories, or entire legal case files in a single call. The independent verification caveat is real; extraordinary efficiency claims require extraordinary evidence. But the architecture is genuinely novel enough that if the numbers are confirmed, this changes the context-window roadmap industry-wide.

### Xiaomi MiMo Code: 20% SWE-Bench Gain via Parallel Planning

Xiaomi open-sourced MiMo Code, a coding agent that generates five parallel action plans at each step, then uses a judge model to select the best before execution ("Max Mode"). The approach improved performance by up to 20% on SWE-Bench Pro at the cost of 4–5× more tokens per session. Available on GitHub under an open license.

**Why it matters:** MiMo Code represents a different strategy for improving agent performance: run inference more times, compare results, pick the best — rather than waiting for a more capable base model. The 4–5× token overhead is significant but workable for high-value coding tasks where correctness matters more than marginal cost. It's also a meaningful signal that hardware companies like Xiaomi are building and open-sourcing serious frontier AI research, not just buying access to Western models.
