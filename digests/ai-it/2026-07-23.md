## 🚀 Моделі

**Microsoft [випустили Mage](https://github.com/microsoft/Mage)** — родину компактних 4B-мультимодальних моделей, які за заявою компанії тягаються з набагато більшими

Mage-Flow генерує і редагує зображення за інструкціями, нативна роздільність 512–2048px, три варіанти інференсу: Base (30 кроків), RL-aligned (20) і дистильований Turbo (4 кроки). За словами Microsoft, за якістю впритул підбирається до Qwen-Image (20B) і FLUX.2 (32B), маючи в 5-8 разів менше параметрів — завдяки VAE-токенізатору, який економить ~12x/22x MAC на піксель при кодуванні/декодуванні.

Другий член сімʼї, Mage-VL (розуміння зображень і відео з проактивним стрімінгом), поки лише анонсований — код "скоро".

MIT-ліцензія, але тільки для дослідницького використання. Чекпоінти Mage-Flow вже на Hugging Face. Якщо потрібна легка модель генерації й редагування зображень без хмарного API — варто зважити проти звичних важковаговиків.

## 🛠️ Інструменти

**Anthropic [відкрили Claude Security](https://code.claude.com/docs/en/claude-security)** для всіх платних планів Claude Code — мультиагентний сканер вразливостей просто в сесії

Команда Claude-агентів мапить архітектуру, будує модель загроз, шукає вразливості і незалежно перевіряє кожну знахідку, перш ніж писати звіт. Патчі — окремим кроком і застосовуються тільки вручну через `git apply`, ніколи автоматично.

```
/plugin install claude-security@claude-plugins-official
/reload-plugins
/claude-security
```

Потрібен Claude Code 2.1.154+ і `python3` 3.9.6+ на PATH (сам плагін нічого не встановлює — юзає тільки стандартну бібліотеку). Можна сканувати весь репозиторій, тільки diff гілки чи окремий коміт; звіти лягають у `CLAUDE-SECURITY-<timestamp>/` з власним `.gitignore`, щоб випадково не потрапили в коміт.

Це вже офіційна фіча з документації, не research preview. Якщо тримаєте приватні репозиторії на GitLab/Bitbucket чи за закритою мережею — цей плагін дотягується туди, куди керований Enterprise-сервіс не може.

#tools

**Claude Code [оновився до 2.1.218](https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md)** — `/code-review` тепер ганяється у фоновому субагенті

Головне: `/code-review` більше не захаращує розмову — виконується в бекграунді, а стек команд лишається ціллю рев'ю. З фіксів: акцидентальне видалення розмови стрілкою вліво (тепер питає підтвердження, а Esc повертає до фонової сесії), побиття Windows-шляхів із `\u`-сегментами (типу `C:\Users\unicorn`) у CJK-символи, і втрату multi-line paste в терміналах, що кодують перенос рядка як Ctrl+J.

Ще: `/mcp` і `claude mcp list` тепер показують HTTP-статус, коли сервер не конектиться; `/ultrareview` приймає описові аргументи типу "review my auth changes" замість падати; skills з `context: fork` тепер за замовчуванням теж ганяються у фоні.

Не такий гучний реліз, як фікс квадратичного уповільнення на днях, але якщо активно юзаєте `/code-review` чи довгі сесії — набір дрібних фіксів приємний.

**[opencodex](https://github.com/lidge-jun/opencodex)** — локальний проксі, який дозволяє Codex CLI ганяти на Claude, Gemini, Grok та 40+ інших провайдерах

Перекладає Responses API Codex у формат інших провайдерів і назад — стрімінг і виклики тулів працюють в обидва боки.

```bash
npm install -g @bitkyc08/opencodex
ocx init
ocx start
codex -m "anthropic/claude-opus-5" "your prompt"
```

MIT-ліцензія. Сам не тестив і не можу перевірити заявлену популярність проєкту — але README і команди реальні.

Якщо звикли до Codex CLI, але хочеться підключити Claude чи локальну Ollama-модель без очікування офіційної підтримки — виглядає як швидкий спосіб спробувати.

## 💡 Рецепти

**LangChain [випустили skill](https://www.langchain.com/blog/towards-automating-eval-engineering)** для Claude Code і Codex, який сам генерує evals з коду агента і його трейсів

Скіл читає структуру репозиторію (промпти, моделі, тули, хуки), майнить патерни з трейсів виконання, якщо вони є, і пропонує, які здібності варто протестувати — а потім питає, які тули запускати живими, а які симулювати. Результат лягає у Harbor-формат: `evals/<task-id>/{task.toml, instruction.md, environment/, tests/}`.

```bash
npx skills add langchain-ai/langchain-skills --skill eval-engineering --yes
```

Після встановлення просто скажіть агенту:
> "Use the eval-engineering skill to create an eval with me. Inspect the agent first, propose a few abilities worth testing, recommend one, and wait for me to choose."

Офіційний реліз від LangChain, а не community-скіл, вийшов 22 липня. Якщо evals для власного агента досі пишете вручну — точка старту, яка хоч якось систематизує процес.

## 📌 Рухи й безпека

**PyPI [заборонив](https://blog.pypi.org/posts/2026-07-22-releases-now-reject-new-files-after-14-days/)** заливати нові файли в релізи старші за 14 днів

Ідея проста: якщо токен публікації чи CI-workflow скомпрометують, зловмисник більше не зможе непомітно підмінити файли у давно стабільному релізі — тільки в свіжому. За даними PyPI, лише 56 із 15 000 найпопулярніших пакетів взагалі публікували файли в релізи старші 14 днів, тож на більшість це майже не вплине.

Привід згадали прямо: минулі інциденти з пакетами LiteLLM і Telnyx.

Якщо тримаєте власний пакет на PyPI і маєте автоматизацію донакатки файлів у старі релізи (рідкісний, але буває кейс) — варто перевірити, що вона переживе цей апдейт.
