## 🛠️ Інструменти

**Claude Code [оновився до 2.1.214](https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md) — фікс дірки в перевірці дозволів на Windows і "стоп-кран" для зловживань**

Пофіксили обхід перевірки дозволів у PowerShell 5.1 на Windows, а Bash-аналіз дозволів тепер краще розбирає редіректи файлових дескрипторів і команди довші за 10 тисяч символів. З нового — інструмент EndConversation: Claude тепер сам може обірвати сесію, якщо вона стає відверто зловживаючою.

Ще додали періодичний heartbeat для довгих tool calls (щоб не виглядало, ніби агент завис), permission-запит на docker-команди з daemon-redirect флагами, і виправили кілька крашів у стрімінгу й settings-файлах.

Дрібний, але щільний реліз — якщо тримаєте Claude Code на автопілоті довго без нагляду, апдейт варте того.

**GitHub Copilot code review [тепер читає CLAUDE.md](https://github.blog/changelog/2026-07-17-copilot-code-review-customization-and-configurability-improvements) і вмикає мережевий firewall за замовчуванням**

Раніше під Copilot доводилось заводити окремі інструкції — тепер він підхоплює `CLAUDE.md`, `GEMINI.md` чи `REVIEW.md`, якщо вони вже лежать у репо. Кастомні інструкції тепер читаються з head-гілки PR, а не з base — можна тестувати їх у фіча-гілці, не вливаючи в main наперед.

Мережевий доступ для code review тепер за замовчуванням обмежений firewall'ом (окремо від інших Copilot-агентів), і з'явився свій `copilot-code-review.yml` для налаштування середовища перевірки.

Якщо репо вже має `CLAUDE.md` під Claude Code — Copilot review підхопить ті самі правила без дублювання інструкцій.

## 💡 Рецепти

**Рецепт: [спальний Mac під повний контроль Claude Code](https://ykdojo.github.io/claude-controls-mac/) — SSH, ізольований акаунт, computer use**

Гайд ставить окремий Mac (без Apple ID, з чистим акаунтом) як "пісочницю", якою Claude Code керує повністю — тільки він, ніяких особистих даних поруч. Доступ — по SSH з основної машини чи навіть з телефону.

```bash
ssh-keygen -t ed25519
ssh-copy-id <user>@<target-host>.local

# не давати маку заснути
sudo pmset -c sleep 0
sudo pmset -c disablesleep 1

ssh <user>@<target-host>.local 'curl -fsSL https://claude.ai/install.sh | bash -s -- 2.1.201'
```

Опційно в репо є ще й скрипт для computer use по SSH і керування браузером через Chrome-розширення.

Ідея проста і чесна: якщо хочете лишати агента без нагляду надовго, дешевше ізолювати його на окремій залізяці, ніж покладатись на permission-систему на основній машині. Гайд свіжий, автор ділиться власним сетапом, на HN уже 33 бали.
