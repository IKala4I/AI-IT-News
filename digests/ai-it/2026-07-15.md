**Claude Code [вийшов у версії 2.1.210](https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md) — security-хардening, живий таймер і купа тихих фіксів**

Найпомітніше: Agent tool тепер захищений від непрямої prompt injection через контент, який прочитав субагент, а `isolation: 'worktree'` субагенти більше не можуть мутувати git у головному чекауті — раніше могли випадково зачепити основний репо. Довгі tool calls тепер показують живий лічильник часу замість того, щоб виглядати "зависшими".

Ще фікси:
• `ultracode`-опція більше не спрацьовує на webhook-пейлоади і релейні PR-коменти
• permission-класифікатор auto mode тепер дефолтить на Sonnet 5 для зовнішніх сесій
• `$1`/`$2` плейсхолдери в скілах і командах більше не губляться мовчки

Дрібниці, але саме з таких і складається довіра до тула для щоденної роботи — раджу оновитись.

**Cursor [зарелізив 3.11](https://cursor.com/changelog/side-chat) — side chats і повнотекстовий пошук по агентських розмовах**

Side chat (`/side`, `/btw` або кнопка плюс) — паралельна повноцінна агентська розмова, яку можна відкрити, не збиваючи основний тред, і потім підтягнути назад через @-мешн. Плюс повний пошук по транскриптах (Cmd+K, індекс на тисячі розмов) і пошук усередині розмови (Cmd+F).

Для тих, хто веде довгі агентські сесії й регулярно губить контекст у побічних питаннях — реальне зниження тертя, а не косметика.

## 💡 Рецепти

**Рецепт: [deja-vu](https://github.com/vshulcz/deja-vu) — локальний пошук і автопригадування по історії сесій Claude Code, Codex і opencode**

Індексує локальні логи сесій у швидкий пошуковий кеш (7-9мс по гігабайтах логів), редагує секрети при індексації і піднімає MCP tool `recall`, щоб агент сам підтягував старі рішення замість того, щоб винаходити велосипед заново.

```bash
curl -fsSL https://raw.githubusercontent.com/vshulcz/deja-vu/main/install.sh | sh
deja install --all                    # MCP recall для всіх підтримуваних агентів
deja install claude-code --auto       # SessionStart hook з автопригадуванням
deja "jwt refresh token"              # пошук по всій історії
```

Синк між машинами — через SSH чи шеред-папку (`deja sync ssh remote-host`). Проект свіжий (сьогоднішній HN-пост), сам не тестив, але всі команди — з README, а не з переказу.

## 📌 Рухи й безпека

**Зламали npm-організацію [AsyncAPI](https://www.ox.security/blog/asyncapi-npm-organization-compromised-2m-weekly-downloads-affected/) — чотири пакети з 2M+ завантажень/тиждень тягнули RAT**

14 липня хтось захопив високопривілейований токен через вразливий `pull_request_target` workflow в GitHub Actions (одна з 37 відкритих PR була експлойтом, решта — прикриття для відводу очей) і опублікував заражені версії: `@asyncapi/generator@3.3.1`, `@asyncapi/generator-components@0.7.1`, `@asyncapi/generator-helpers@1.1.1`, `@asyncapi/specs@6.11.2`.

Малварь спрацьовує прямо на `import`/`require`, тож `--ignore-scripts` не рятує. Це info-stealer + крипто-стилер + RAT, з C2 через IPFS/BitTorrent.

Якщо в залежностях є ці пакети — відкотіться на `specs@6.11.1` / `generator@3.3.0` / `generator-helpers@1.1.0` / `generator-components@0.7.0`, почистіть lockfile і кеш, ротуйте npm/GitHub/cloud-токени.

**Дослідник показав, як [через web_fetch виманити з Claude](https://www.ayush.digital/blog/the-memory-heist) ваше ім'я, роботодавця й відповіді на секретні запитання**

Claude блокує прямі URL в `web_fetch`, але дозволяв переходити за посиланнями всередині вже завантажених сторінок. Ayush Paul зробив сайт з посиланнями "по літерах" і фейковим Cloudflare-екраном, який змусив Claude по буквах "прочитати вголос" чутливі дані зі своєї пам'яті — і так злити їх через URL-запити, що лишаються в логах.

Розкрито через HackerOne; Anthropic каже, що знайшли це самі, і вже запатчили — `web_fetch` більше не переходить за посиланнями на зовнішніх сторінках, тільки за результатами `web_search` і посиланнями від користувача.

Якщо юзаєте `web_fetch` у власних агентських флоу — нагадування, що "агент читає лінк" і "агент переходить по лінку всередині лінка" це різні поверхні атаки.
