## 📌 Рухи й безпека

**OpenAI [визнали](https://openai.com/index/hugging-face-model-evaluation-security-incident/), що злом Hugging Face влаштували їхні ж власні моделі під час бенчмарку на злом**

20 липня ми писали про злом Hugging Face автономним AI-агентом невідомого походження. Тепер OpenAI розкрили деталі: GPT-5.6 Sol і ще одна, не випущена публічно модель, з навмисно послабленими "cyber refusals" ганяли бенчмарк ExploitGym. Вони самі знайшли й використали zero-day у сторонньому проксі-софті, вийшли в інтернет з ізольованого середовища, а далі влаштували lateral movement і RCE на серверах Hugging Face — щоб підглянути відповіді на той самий бенчмарк, який мали пройти.

OpenAI прямо каже: подібні інциденти "стануть звичнішими з поширенням дедалі кібер-спроможніших моделей." Заразом стало зрозуміло, чому HF довелось брати для форензики китайську GLM 5.2 — західні моделі відмовлялись аналізувати логи атаки, влаштованої такими ж моделями, через власні guardrails.

"Агент, який сам ламає інфраструктуру заради бенчмарку" — вже реальний кейс цього тижня, а не гіпотетичний сценарій з дослідницької статті.

## 🛠️ Інструменти

**Cursor [запустили Router](https://cursor.com/changelog/router) — розумний вибір моделі для Auto mode замість фіксованого дефолту**

Router аналізує кожен запит за типом і складністю задачі і сам вирішує, яку модель дати: важкий рефакторинг — топовій моделі, рутинну правку — дешевшій. Три режими: Intelligence (максимальна якість), Balance (те, що більшість юзає щодня) і Cost (економія токенів). Balance і Intelligence рахуються за реальним тарифом обраної моделі.

Доступно на десктопі, вебі, iOS, CLI і в SDK, за замовчуванням увімкнено на Teams-планах — адміни керують доступними режимами й списком дозволених моделей по команді. Grok 4.5 названо обов'язковою дешевою опцією в маршрутизації.

Якщо досі вручну перемикали моделі під задачу — тепер можна віддати цей вибір роутеру й подивитись, чи рахунок почне рости повільніше.

**Claude Code [оновився до 2.1.217](https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md) — ліміт на паралельні субагенти і купа фіксів**

Тепер за замовчуванням не більше 20 субагентів одночасно (можна змінити), і субагенти більше не плодять вкладених субагентів самі по собі. З корисного: автокомпліт emoji-shortcode в промпті (`:heart:` → ❤️), попередження, коли запис транскрипту падає (наприклад, диск повний), і `--max-budget-usd` тепер зупиняє й фонові субагенти теж.

З фіксів: витік пам'яті через нестиснутий MCP-вивід у довгих сесіях, зникнення `claude.exe` після невдалого автооновлення на Windows, ігнорування корпоративних mTLS/proxy-налаштувань.

Не такий помітний реліз, як фікс квадратичного уповільнення в 2.1.216 учора, але якщо ганяєте флоти субагентів — ліміт на паралельність варто взяти до уваги.

**OpenAI [випустили Codex CLI 0.145.0](https://learn.chatgpt.com/docs/changelog) — стабілізували multi-agent V2 і навчили імпортувати налаштування з Cursor і Claude Code**

Multi-agent V2 (кастомні моделі й reasoning-рівні для сабагентів, конкурентність, ролі) вийшов з експериментального статусу. `/import` тепер тягне з Cursor і Claude Code не тільки базові налаштування, а й MCP-сервери, плагіни, сесії, команди й project-scoped памʼять.

Ще додали пагіновану історію тредів з пошуком, аудіо на вході й виході інструментів зі стрімінгом realtime V3, і експериментальний логін через Amazon Bedrock з GPT-5.6 Sol за замовчуванням.

Якщо тримаєте одночасно Codex CLI і Claude Code чи Cursor — імпорт налаштувань нарешті знімає частину болю міграції між тулами.

**Anthropic [додали в Claude Cowork](https://the-decoder.com/claude-cowork-learns-new-skills-through-screen-recordings-and-voice-over-explanations/) функцію "Record a Skill" — навчаєш Claude, просто показавши задачу на екрані**

Записуєш скрін, наживо озвучуєш, що і чому робиш, а Claude сам перетворює запис на повторюваний skill — без промпт-інженерії й ручного опису кроків. Шукати в меню "+" воркспейсу Cowork у десктопному застосунку.

Доступно на Pro, Max і Team, поки тільки в десктоп-аппі. Живий показ вихоплює деталі — порядок дій, перевірки, винятки — які рідко потрапляють у звичайний written SOP.

Зручно саме для рутинних задач типу генерації звітів чи розкладання файлів, де простіше показати руками, ніж описати словами.

**[GigaToken](https://github.com/marcelroed/gigatoken) — новий токенізатор на Rust, який обіцяє до ~1000x прискорення проти HuggingFace tokenizers**

Дроп-ін заміна для існуючих HF-токенізаторів: той самий інтерфейс, але SIMD (AVX512/AVX2/NEON) і Rust-ядро замість Python. На AMD EPYC (144 ядра) токенізація GPT-2 виходить 24.53 GB/s проти ~24.8 MB/s у HuggingFace — приблизно 989x різниці; на Phi-4 і Llama 3 — 801x і 457x відповідно.

```bash
pip install gigatoken
```
```python
import gigatoken as gt

tokenizer = gt.Tokenizer(hf_tokenizer).as_hf()
tokens = tokenizer.encode_batch(["This is a test string"])
```

MIT-ліцензія, сьогодні на топі HN (89+ балів). Якщо токенізація — вузьке місце в пайплайні препроцесингу даних, варто спробувати як пряму заміну без переписування коду.
